Part A:

  1. The file you will be working with for Part A is an output file from a program called "Breakdancer". For Part A, make a directory called "BD" within your Project3 directory. Copy the BD output file from canvas to this new folder. As with similar files, header lines start with a #. The most important part of the header is this line:

    #Chr1   Pos1    Orientation1    Chr2    Pos2    Orientation2    Type    Size    Score   num_Reads       num_Reads_lib   M_Arctoides.chr6.recalibrated.bam

  2. Breakdancer takes a genome alignment file (bam format) as input and outputs potential structural variants. We will sift through the raw output to identify high quality variants. Specifically, breakdancer identifies five possible structural variants (SVs): Insertions (INS), Deletions (DEL), Inversions (INV), and Translocations (ITX/CTX). The first thing you need to do is get a count of how many of each type of variant is in this file.

    Variant Type Count in File Filtered count (see #4)
    Insertions
    Deletions
    Inversions
    Translocations (ITX)
    Translocations (CTX)

  3. The next thing you need to do is to gain a better understanding of each type of variant. To accomplish this, calculate the mean value for each of the following columns per variant type: Size, Score, num_Reads.

    Special note for Size: The column for size has both negative and positive numbers, making it difficult to calculate a mean value. Also, several of these values are just plain WRONG! Therefore to get the mean, you will need to re-calculate the length of each variant from the Pos1 and Pos2 columns. Depending on the tool you use to do this calculation, you may have issues with it printing, for which you are allowed to use the internet. For translocations between chromosomes, you can ignore the fact that the coordinates are on different chromosomes and do the calculations the same as other types of variants.

    Variant Type Mean Score Mean Size Mean # Reads
    Insertions
    Deletions
    Inversions
    Translocations (ITX)
    Translocations (CTX)

  4. Finally, we will want to filter for high quality variants. Let's repeat the Table we made in #2, but only for filtered variants. First, remove all variants with a score below 95. The next filter we will want to impose is based on the number of reads. In addition to the score filter, now require that our trusted variants are supported by at least 10 reads. Go back and fill in the count for each type based on these two filters combined.

Part B:

  1. For part B of your project, you will be acting as a restriction enzyme. Restriction enzymes cleave DNA based on a recognition sequence. Retrieve the fasta sequence from CANVAS. Make a new directory in your Project3 folder called "Enzyme" and download this zipped fasta file into it. Unzip the file.

  2. Your goal will be to read in the sequence and "cut" the DNA (i.e. insert a newline) everytime you encounter a given cleavage site. Because the purpose of this assignment is to practice your regular expression usage, assume all enzymes cleave with blunt ends. Don't get fancy here folks! You should think carefully about which tools you've learned in class are appropriate for this exercise. It may be easier to first identify the number of cleavage sites within the DNA sample prior to cutting. You may find that more than one tool will work, so go with whichever one you feel most confident using. To avoid getting the number of cut sites wrong, consider removing the fasta header from the sequence file.

  3. One of the most common restriction enzymes is EcoRI, which has a recognition site of GAATTC. Fill in the table below on the number of resulting fragments from EcoR1 restriction enzyme.

    Restriction Enzyme Cleaveage Site Number of Fragments Average Fragment Length
    EcoR1 GAATTC

  4. Now that you have cleaved the larger DNA sequence file into multiple lines, compute the average length of the resulting fragments you made "in silico". You may search the internet for options to do this. Add the result to the table in #3 above.

    Note: the key will allow all values to be off by +/- 1. No partial credit will be awarded for table values - they are either right or wrong. So consider making test files where you KNOW the answer to test that your code works as expected.

  5. Two other restriction enzymes have more variable restriction sites. In DNA, ambiguous bases are indicated via specific codes. As you did above, repeat the process with these two new restriction sites. Unlike EcoRI, the variability will require you to generate a regular expression based on their site ambiguity in order to cleave them. As above, once you cleave them, be sure to calculate the average fragment length as well.

    Restriction Enzyme Cleavage Site Number of Fragments Average Fragment Length
    AcyI GRCGYC
    AhdI GACNNNNNGTC
